# Product

## Register

product

## Users

ผู้ใช้หลักคือพนักงานในองค์กรของมูลนิธิสาธารณสุขแห่งชาติ (NHF) ที่ต้องทำงานประจำวันผ่านระบบเดียวกัน เช่น ดูข้อมูลส่วนตัวและพนักงาน แจ้งปัญหา IT ติดตามสถานะงาน ขออนุมัติวันลา เบิกหรือจัดการสต็อก และรับการแจ้งเตือนที่เกี่ยวข้องกับงานของตน

ผู้ใช้บางกลุ่มมีสิทธิ์ดูแลระบบหรืออนุมัติงาน เช่น HR, IT admin, ผู้จัดการ, ผู้อนุมัติ และผู้ดูแลสต็อก ซึ่งต้องการหน้าจอที่อ่านสถานะได้เร็ว ลดความผิดพลาด และทำงานซ้ำได้อย่างมั่นใจ

## Product Purpose

ระบบนี้เป็นศูนย์กลางงานภายในองค์กรสำหรับจัดการข้อมูลพนักงาน งานบริการ IT ระบบลา ระบบสต็อก การแจ้งเตือน และรายงานที่เกี่ยวข้อง เป้าหมายคือให้พนักงานทำงานบริการภายในได้ง่ายขึ้น ติดตามสถานะได้ชัดเจน และให้ผู้ดูแลระบบจัดการงานหลังบ้านได้เป็นระเบียบ

ความสำเร็จของระบบวัดจากความเร็วในการหาข้อมูล ความชัดเจนของสถานะงาน ความถูกต้องของสิทธิ์การเข้าถึง และจำนวนขั้นตอนที่ผู้ใช้ต้องทำเพื่อจบงานสำคัญแต่ละอย่าง

## Brand Personality

เรียบง่าย สุภาพ ใช้งานง่าย

น้ำเสียงของระบบควรตรงไปตรงมา เป็นมิตร และไม่ใช้ถ้อยคำเกินจริง หน้าจอควรรู้สึกน่าเชื่อถือแบบเครื่องมือทำงานภายในองค์กร ไม่ใช่หน้าโฆษณาหรือผลิตภัณฑ์ที่เน้นความหวือหวา

## Anti-references

ไม่ควรมีหน้าตาแบบ SaaS marketing ที่ใช้ลูกเล่นเยอะ สีจัด ตัวอักษรใหญ่เกินจำเป็น หรือองค์ประกอบตกแต่งที่ไม่ได้ช่วยให้ทำงานง่ายขึ้น

ไม่ควรรู้สึกรกเหมือนระบบองค์กรเก่าที่ข้อมูลแน่นแต่ลำดับความสำคัญไม่ชัดเจน และไม่ควรใช้ motion, เงา, gradient, หรือ card จำนวนมากจนรบกวนงานหลักของผู้ใช้

## Design Principles

1. งานหลักต้องเห็นก่อนเสมอ: ทุกหน้าควรช่วยให้ผู้ใช้รู้ทันทีว่าต้องทำอะไรต่อ สถานะไหนสำคัญ และปุ่มไหนคือการกระทำหลัก
2. ความเรียบต้องไม่แลกกับความคลุมเครือ: ลดสิ่งตกแต่ง แต่คงลำดับชั้นของข้อมูล สถานะ และ feedback ให้ชัดเจน
3. ภาษาไทยต้องอ่านง่ายและเป็นธรรมชาติ: คงคำไทยเดิมเมื่อเหมาะสม ใช้คำสั้น ตรง และไม่เปลี่ยนภาษาโดยไม่จำเป็น
4. ระบบต้องไว้ใจได้สำหรับงานภายใน: สิทธิ์ บทบาท สถานะ และผลลัพธ์หลัง action ต้องสื่อสารชัดเจน ลดโอกาสคลิกผิดหรือเข้าใจผิด
5. ความสม่ำเสมอสำคัญกว่าความแปลกใหม่: ปุ่ม ฟอร์ม ตาราง badge navigation และ dialog ควรใช้ pattern เดียวกันทั้งระบบ

## Accessibility & Inclusion

ตั้งค่าเริ่มต้นตามแนวทาง WCAG AA: คอนทราสต์อ่านง่าย รองรับการใช้งานด้วยคีย์บอร์ด มี focus state ชัดเจน ใช้ข้อความ error และสถานะที่ไม่พึ่งสีอย่างเดียว และเคารพ `prefers-reduced-motion`

ควรให้ความสำคัญกับภาษาไทย ความยาวข้อความจริงในระบบ และผู้ใช้ที่ไม่ได้ใช้ระบบไอทีเป็นงานหลักทุกวัน ดังนั้น UI ต้องสแกนง่าย ไม่ใช้คำกำกวม และไม่ซ่อน action สำคัญไว้หลัง pattern ที่เดายาก
