# รองรับ flow พิเศษของระบบลาโดยไม่เปลี่ยนความหมายของโควต้าเดิม

ระบบลาต้องรองรับสามกรณีพิเศษ: ลาย้อนหลังกรณีฉุกเฉิน, การแจ้งว่าอนุมัติแล้วแต่ไม่ได้ใช้วันลา, และคำขอลาเกินโควต้ากรณีพิเศษ เราตัดสินใจให้กรณีเหล่านี้ยังอยู่ใน flow อนุมัติเดิมของผู้อนุมัติ โดยเพิ่มข้อมูลประกอบและสถานะเฉพาะแทนการแก้โควต้าเงียบ ๆ หรือใช้สถานะ `CANCELLED` ร่วมกัน เพราะต้องรักษา audit trail และทำให้รายงานแยกความหมายของ “ยกเลิกก่อนอนุมัติ”, “ไม่ได้ใช้วันลา”, และ “ใช้เกินสิทธิ์” ได้ชัดเจน

คำขอเกินโควต้าจะทำให้ยอดใช้ปีปัจจุบันเกินโควต้าได้ และแสดงเป็นเกินสิทธิ์แทนการเพิ่ม `totalDays` หรือยืมโควต้าปีหน้า เพื่อให้ยังเห็นความแตกต่างระหว่างสิทธิ์ตามนโยบายกับการอนุมัติกรณีพิเศษอย่างชัดเจน
